You'll enter the VM's password once. The VM must have Remote Login on (System Settings → General → Sharing). This command fixes the ~/.ssh permissions that often block ssh-copy-id, and skips host-key fingerprint prompts / known_hosts entries (so reused VM IPs never cause conflicts — the same way Tart Oven connects).


      
3 · Point Tart Oven at the key

      
Set SSH identity file above to ~/.ssh/tart-oven, then click Save configuration. Test it with Get info on a running VM.


      
Tip · Do it once for the whole fleet

      
Add the public key to your TEMPLATE VM's ~/.ssh/authorized_keys before cloning — every clone inherits it, so you never repeat steps 1–2.
